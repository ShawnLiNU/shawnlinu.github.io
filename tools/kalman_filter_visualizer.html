The Kalman filter is an optimal recursive estimator. At each step it combines two sources of information:

• Prediction — where the model says the state should be.
• Measurement — what the noisy sensor reports.

It blends them using the Kalman gain K, which is computed automatically from Q and R. Watch K stabilize as the filter converges — this is the steady-state gain.
Predict
x̂− = F x̂
P− = F P FT + Q
Update
K = P−HT(H P−HT + R)−1
x̂ = x̂− + K(z − H x̂−)
P = (I − KH) P−
In 1D: F=1, H=1, so K = P/(P+R) — a simple weighted average between prediction and measurement.
Q too low — filter is overconfident in its model. Estimate lags real changes, slow to react.

Q too high — filter trusts every measurement. Estimate is noisy, tracks jitter.

R too low — filter trusts a bad sensor. Estimate oscillates with noise.

R too high — filter ignores sensor. Estimate drifts with model errors.

Optimal: set Q ≈ variance of the true process disturbance, R ≈ variance of sensor noise.
